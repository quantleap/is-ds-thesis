Administrator Entity Deduplication and Resolution¶

The CIR register publishes administrator by name and office address. These data fields are free text fields and contain many typos and alternative ways of writing. We would like to uniquely define a administrator and link the insolvency cases to this entity.

The chosen golden standard for the administrator entity is the Bar/Tableau where lawyers are registered, maintained by the Nederlandse Orde van Advocaten (NOvA).

To resolve CIR administratore to NOvA lawyers we: First we deduplicate administrator names by following a number of normalization rules.
Secondly we resolve the normalized name to the most relevant lawyer found in the NOvA register

For results we check the amount of deduplication occurring, the recall of lawyers and the precision.

Deduplication by normalization of administrator name¶

initials¶

  • make lowercase and strip
  • replace commas with periods
  • make double names into single names (e.g. albert-jan: a-j. -> a.j.)
  • make double character name into single character name (e.g. theodor: th. -> t.)
  • ensure all characters are followed by a period

middle part¶

  • move middle part to family name (prevent double occurence)

family name¶

  • make lowercase and strip
  • remove accents
  • remove titles (jhr, baron, llm, mba, etc)
  • remove initials
  • remove CIR specific comments
  • remove periods
  • remove spaces around hyphens

Results¶

Adding maiden name removal to the normalization function reduces the amount of names further with 53/1459 = 3.6%. But the register names to which the normalized names are resolved carry the maiden name. Therefor we choose to keep the maiden name as it brings extra information to the resolution function.

Administrator Entity Resolution¶

We do not have an complete lawyer set to match the normalized name against.

If a normalized names has no exact match we:

  • [do a fuzzy match against the master data records] and if score is good use the match
  • use the NOvA online register to find a match

The NOvA register is searchable but instead of scraping the entire register of lawyers we use NOvA's search functionality with the application of filters to come up with a best match.

Instead of using the web form we can directly use the underlying JSON REST API to find the corresponding lawyer. We start with the most precise filter, if no results are found we widen the net with a less precise filter in three steps:

  • step 1: filter on specialty curator,
  • step 2: filter branch insolvency law,
  • step 3: no filter,

if multiple results are returned we pick the most relevant. NOvA supplies relevancy on name and total query.

Issues:

  • queries on simple terms, still return 100+ results.

The two lawyer entities, cir administrator and NOvA lawyer are joined together in an association table where an exact match can be looked up when needed without going to the NOvA webservice. The association table can also be post-processed or manually altered to reach a 100% resolution.
